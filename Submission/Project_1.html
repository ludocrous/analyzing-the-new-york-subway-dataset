Section 1 - Statistical Test¶

The NYC Subway data is presented as a file of observations with the following variables:

  • UNIT - The station entry unit
  • DATEn - Date of observation
  • TIMEn - Time of observation
  • Hour - A numeric from 0-23 indicating the hour of day
  • DESCn - The type of entry REGULAR, etc
  • ENTRIESn_hourly - Number of entries since previous REGULAR observation
  • EXITSn_hourly - Number of exits since previous REGULAR observation
  • maxpressurei - Maximum pressure for that day (inches Hg)
  • maxdewpti - Maximum dew point for that day (Deg F)
  • mindewpti - Minimum dew point for that day (Deg F)
  • minpressurei - Minimum pressure for that day (inches Hg)
  • meandewpti - Average dew point for the day
  • meanpressurei - Average pressure for the day
  • fog - 0 indicating no fog and 1 fog
  • rain - 0 indicating no rain and 1 rain
  • meanwindspdi - Average windspeed in mph for that day
  • mintempi - Minimum temperature for that day (Deg F)
  • meantempi - Average temperature for that day (Deg F)
  • maxtempi - Maximum temperature for that day (Deg F)
  • precipi - Precipitation in inches at that time
  • thunder - 0 for no thunder activity and 1 for thunder

Shapiro-Wilk test p-values for normality - Rain Days: 0.00000 - Non-Rain Days: 0.00000

As the p-value returned from is extremely small, we reject the null hypothesis and conclude that neither sample appears to be from a Normally Distibuted population.

This makes Welch's t-test is inappropriate, and we perform a non-parametric Mann-Whitney U test instead.

Because either sample could produce larger results we perform a two-tail Mann-Whitney with an level = 0.05 We use the typical null-hypothesis for Mann-Whitney which stipulates that the probability of drawing a random sample from the first population (which is our sample without rain) being larger than a random sample from the second population (in this case our sample with rain) is 0.5. i.e. the populations are identical.

Because the models return p-values for a 1-tailed test we are required to double the result for a 2-tailed test

def day_of_week(date,return_string = False):
    '''
    Return the day of the week as in integer with Monday=0 and Sunday=6
    '''
    timeStruct = time.strptime(date,"%Y-%m-%d")
    #dateStr = time.strftime("%Y-%m-%d",timeStruct)
    #date_formatted = dateStr
    if return_string:
        return ["0-Monday","1-Tuesday","2-Wednesday","3-Thursday","4-Friday","5-Saturday","6-Sunday"][timeStruct.tm_wday]
    else:
        return timeStruct.tm_wday

def normalize_features(df):
    """
    Normalize the features in the data set.
    """
    mu = df.mean()
    sigma = df.std()
    
    if (sigma == 0).any():
        raise Exception("One or more features had the same value for all samples, and thus could " + \
                         "not be normalized. Please do not include features with only a single value " + \
                         "in your model.")
    df_normalized = (df - df.mean()) / df.std()

    return df_normalized, mu, sigma

def predictions_OLS(dataframe):
    values = dataframe['ENTRIESn_hourly']
    m = len(values)

    # Add weekday number and is_weekend boolean features
    dataframe["day_of_week"] = dataframe["DATEn"].map(lambda x: day_of_week(x))
    dataframe["is_weekend"] = dataframe["day_of_week"].map(lambda x: 1 if (x>=5) else 0)
  
    # Select Features (try different features!)
    features = dataframe[['rain', 'Hour', 'fog', 'mintempi', 'is_weekend']]
    
    # Add UNIT to features using dummy variables
    dummy_units = pd.get_dummies(dataframe['UNIT'], prefix='unit')
    features = features.join(dummy_units)

    features, mu, sigma = normalize_features(features)
    features['ones'] = np.ones(m) # Add a column of 1s (y intercept)
    
    values_array = np.array(values)
    model = sm.regression.linear_model.OLS(values_array,features,missing='none',hasconst=True)
    
    return model

What features (input variables) did you use in your model? Did you use any dummy variables as part of your features?

Before performing regression we modified the dataset to add two new features:

  • day_of_week: This is set to 0 for a Monday through to 6 for a Sunday and derived from DATEn
  • is_weekend: This is a boolean set to 0 if day_of_week in (0,1,2,3,4) and 1 if in (5,6)

We also create dummy variables for each value of UNIT, which has the disadvantage of introducing numerous variables to the regression model, but dramatically improves the results.

The final selection of features (excluding dummies) was : 'rain', 'Hour', 'fog', 'mintempi', 'is_weekend'

def compute_r_squared(data, predictions):
    # Write a function that, given two input numpy arrays, 'data', and 'predictions,'
    # returns the coefficient of determination, R^2, for the model that produced 
    # predictions.
    # 
    # Numpy has a couple of functions -- np.mean() and np.sum() --
    # that you might find useful, but you don't have to use them.

    # YOUR CODE GOES HERE
    data_mean = np.mean(data)
    obs_exp_diff = data - predictions
    obs_mean_diff = data - data_mean
    r_squared = 1 - (np.dot(obs_exp_diff,obs_exp_diff)/np.dot(obs_mean_diff,obs_mean_diff))
    
    # Could also use numpy.square and then numpy.sum() to square the array but retain its size
    # symbol to square values of the array appears to be **2 

    return r_squared
